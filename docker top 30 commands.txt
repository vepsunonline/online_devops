Docker Image Commands
---------------------------
1 to see the list of all docker images 
	#docker images or #docker image ls

2. To download an image from dockerhub
	#docekr pull image_name

3. To Upload an into docker hub
	#docker push image_name

4. To remove an image 
	#docker rmi image_name

5.To build an image from a docker file
	#docker build -t image_name .

Note: . represent current workig directory

6. To create an image for a container
	#docker commit container_name/cont_id     image_name

7. To remove all images that are not running as containers
	#docker images -f dangling=true

Commands to work on containers 
-----------------------------------
8. to see the list of all running containers
	#docker container ls

9. to seee the list of all containers (running and stopped)
	#docker ps -a

10. to stop a runnning container
	#docker stop container_name/cont_id

11. to start a stopped container
	#docker start cont_name/cont_id

12. to restart a container
	#docker restart container_name/cont_id
	to restart a container after 15 seconds 
	#docker restart -t 15 cont_name/cont_id

13. to delete a stopped container
	#docker rm cont_name/cont_id

14. to delete a running container
	#docker rm -f cont_name/cont_id

15. to stop all running containers
	#docker stop $(docker ps -aq)

16. to remove all stopped containers
	#docker rm $(docker ps -aq)

17. to remove all containers (running and stopped)
	#docker rm -f $(docker ps -aq)

18. To Remove all docker containers, networks, images
	#docker system prune -a

19. To start an image as container and also pull an image and start it 
	#docker run image_name
	run command options
	-----------------------------------------	
	--name 		used for giving a name for a container
	-it		used to open an interactive terminal in the container
	-d		used to run container in detached mode as a deamon
	-v		used to attach volumes
	--volumes-from 	for creating and attaching reusables volumes
	--network		Used for attaching a network to a container
	-p		Used for port mapping Eg: -p 8080:80 will assign the continer port 80 to docker host port 8080
	-P 		Used to publish the port numbers. The internal port of the docker container will be mapped with a port greater than 30000
	-e		Used for specifing the environment variables of a container
	--cpu		used to specify the processors that the container should use
	-rm		this will delete the container once we exit from it
	-m		used for limiting the amount of memory that a container can use

20. To find the detailed info about a container
	#docker inspect container_name/container_id

21. To see the logs that are generated by a container 
	#docker logs container_name/cont_id

22. To open interactive terminal in a container in which the bash shell is not running
	#docker exec container_name/cont_id bash

23. To reopen bash in a container which is running in background
	#docker attach cont_name/cont_id

24. To come out of a container without exit
	ctrl+p, ctrl+q

Docker Swarm Creating Replicas
---------------------------------------------------
37. To initiate docker swarm
	#docker swarm init

38. To join docker swarm  master
	#docker swarm init --advertise-addr  "master ip-addr"

39. To exit from the docker service 
	#docker swarm leave

40. To list the nodes
	#docker nodes ls

41.  to create create replicas
	#docker service create --replicas 5 -p 80:80 --name web nginx

42. to check the list of services
	#docker service ls

43. Command to check how many replicas running for a service in diff nodes
 
	#docker service ps web 	or 	#docker service ps

44. to increase/decrease the no of replicas
	#docker service scale web=10
